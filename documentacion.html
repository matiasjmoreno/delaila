get /pedidos
Accede a todos los pedidos (pedidosGet)
Trae una lista con todos los pedidos registrados en la base de datos. Por cada pedido, devuelve tantos registros como productos diferentes posea.

Request headers

Return type

array[PedidoAdmin]

Example data

Content-Type: application/json
[ {
  "estado" : "En Preparación",
  "total" : 440,
  "usuario_id" : 4,
  "hora" : "2020-05-20T00:48:08.000Z",
  "direccion" : "Av Colon 4800",
  "nombre_producto" : "Hamburguesa",
  "id" : 1,
  "cantidad" : 2,
  "metodo_pago" : "Débito"
}, {
  "estado" : "En Preparación",

patch /pedidos/{id}
Edita el estado de un pedido. (pedidosIdPatch)
Dado el id de un pedido y un nuevo estado, se actualiza el registro en la base de datos.

Path Parameter — ID del pedido a actualizar

Consumes

This API call consumes the following media types via the Content-Type request header:

Request body

Body Parameter — Nuevo estado del pedido

Request headers

Responses

post /login
Autentica usuarios (loginPost)
Dado un nombre de usuario y una contraseña, corrobora los datos con la base de datos. De existir el usuario y estar bien los datos, devuelve el token necesario para futuras autenticaciones.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body UsuarioLogin (optional)
Body Parameter — Nombre de usuario y contraseña

Return type

inline_response_200

Example data

Content-Type: application/json
{
  "token" : "IUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c3VhcmlvX3Rva2VuIjp7Im"
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Token para futuras autenticaciones inline_response_200

400

Error inline_response_400_1

post /pedidos
Crea un nuevo pedido (pedidosPost)
Dados los datos requeridos, realiza un nuevo pedido.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body PedidoUsuario (optional)
Body Parameter — Los productos se informan en forma de lista con los ID de los productos pedidos por el usuario, acompañado de un array de las cantidades de los mismos. Las posiciones deben corresponderse. Es necesario además aclarar la dirección de entrega y el método de pago.

Responses

200

El pedido fue realizado con éxito

401

Primero debes iniciar sesión

403

Token inválido
